30-day data science learning roadmap for intermediate learners

Week 1 milestone

Advanced data wrangling & feature engineering

Daily tasks

Day 1–2Advanced pandas: multi-index, groupby chains, window functions
Day 3–4Feature engineering: encoding, binning, interaction terms
Day 5–6Handling missing data & outliers with robust techniques
Day 7Review + build a clean reusable data pipeline class

Resources

pandas docs Kaggle pandas sklearn preprocessing Python for Data Analysis

Week 1 project

Real-world dataset pipeline

Take a messy Kaggle dataset (e.g. NYC taxi or Titanic extended). Build a full ingestion → cleaning → feature-engineering pipeline with unit tests.

Week 2 milestone

Advanced ML models & model selection

Daily tasks

Day 8–9Ensemble methods: gradient boosting (XGBoost, LightGBM, CatBoost)
Day 10–11Hyperparameter tuning: Optuna, RandomizedSearchCV, early stopping
Day 12–13Cross-validation strategies: stratified, time-series, group k-fold
Day 14Model stacking & blending — build a meta-learner

Resources

XGBoost docs Optuna Kaggle competitions LightGBM

Week 2 project

Kaggle competition entry

Join a tabular Kaggle competition. Train XGBoost & LightGBM models, tune with Optuna, and submit a stacked ensemble — aim for top 40% leaderboard.

Week 3 milestone

Deep learning fundamentals & NLP/CV basics

Daily tasks

Day 15–16PyTorch essentials: tensors, autograd, custom datasets & dataloaders
Day 17–18CNNs for image classification — train ResNet on a custom dataset
Day 19–20Transformers & BERT: fine-tune a HuggingFace model for text classification
Day 21Model interpretability: SHAP, LIME, attention visualization

Resources

PyTorch tutorials HuggingFace course fast.ai SHAP docs

Week 3 project

Sentiment analysis API

Fine-tune a BERT model on a domain-specific sentiment dataset (e.g. product reviews). Expose it as a FastAPI endpoint with SHAP-based explanations.

Week 4 milestone

MLOps, deployment & capstone project

Daily tasks

Day 22–23MLflow: experiment tracking, model registry, artifact logging
Day 24–25Containerize model with Docker; deploy on Hugging Face Spaces or Render
Day 26–27Data drift monitoring with Evidently AI; set up alerts
Day 28–30Capstone: end-to-end project — problem → model → live demo

Resources

MLflow docs Docker docs Evidently AI FastAPI

Capstone project

End-to-end ML system (portfolio piece)

Pick a real problem (churn prediction, image tagger, news classifier). Build the full pipeline: data → model → API → deployed app with drift monitoring. Document it on GitHub with a writeup.

Final outcome after 30 days

A deployed ML portfolio project, a Kaggle leaderboard entry, a fine-tuned NLP model, and working knowledge of production MLOps — ready to apply for mid-to-senior data scientist roles.
